# 카누(KANU) 캡슐 업데이트

카누 공식몰 API(`scripts/kanu-fetch.sh`)로 3개 라인의 캡슐을 가져와 `src/data/capsules.json`을 최신화한다.

| 라인 | 카테고리 | 앱 compat | nameKo 접미어 |
|------|:--:|--------|--------|
| 바리스타 전용 | 930896 | `kanubarista` | (없음) |
| 네스프레소 오리지널 호환 | 930897 | `original` | ` (네스프레소 호환)` |
| 돌체구스토 호환 | 930898 | `dolcegusto` | ` (돌체구스토 호환)` |

## ⚠️ 카누 API의 한계
카누 API는 **강도(intensity)와 향미(aromatics)를 제공하지 않는다.** 이름·가격·설명(promotionText)·구매URL만 있다.
- 그래서 **신규 캡슐은 강도 0(미정)** 으로 추가되고, 향미는 설명 문구에서 추정한다.
- **실제 강도·향미·영문명은 카누 공식 "강도 이미지"를 보고 사람이 채워야 한다.** 스크립트가 `[보완 필요]`로 플래그한다.
- 기존 캡슐의 강도·향미·영문명(큐레이션 값)은 절대 덮어쓰지 않는다.

## 정책
- **신규** → 추가(강도 0, 향미는 설명에서 추정, 영문명=한글 임시, 가격·구매URL·설명은 실제값).
- **사라진 캡슐** → `isEnabled: false` (소프트 단종, 이력 보존).
- **다시 전시** → `isEnabled` 해제.
- **기존** → 가격·구매URL만 갱신.
- 브랜드 `카누`만 대상. 다른 브랜드는 건드리지 않는다.
- 매칭은 라인(compat) + 캡슐명 기준이며 "싱글오리진" 접두어 유무는 무시(정규화).

## 실행 절차
1. **미리보기**: `node scripts/update-kanu.mjs --dry-run` → 리포트 검토.
   - ⚠️ 안전장치: 특정 라인 개별 캡슐이 0개면 조회 실패로 보고 스크립트가 중단한다. 단종이 비정상적으로 많으면 원인부터 확인.
2. **반영**: `node scripts/update-kanu.mjs`
3. **신규 캡슐 보완** (`[보완 필요]` 항목):
   - 사용자에게 **카누 공식 강도 이미지**를 요청한다(캡슐별 강도 막대 + 아로마).
   - 이미지 기준으로 `intensity`, `notes`, `name`(영문), 필요시 `acidity/body/bitterness`를 채운다.
   - 강도가 진짜 없는(아이스 등) 캡슐만 0으로 둔다.
4. **검증**: `npm run typecheck`. 필요시 preview.
5. **보고**: 추가/단종/가격변경/보완필요 요약 전달.

## 커밋
- **요청 없이 커밋·푸시하지 않는다.**

## 참고
- 가격은 `salePrice ÷ 팩 캡슐 수`(개당). 번들(컬렉션/세트/버라이어티/팩, 20캡슐 초과)은 제외.
- 새 향미가 필요하면 `src/data/notes.json`에 추가(한글/영문/아이콘). 설명 추정 규칙은 스크립트의 `NOTE_RULES` 참고·확장.
