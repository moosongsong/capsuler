# illy(일리) 캡슐 업데이트

illy는 **공개 JSON API가 없고** PHP 쇼핑몰이 상품을 HTML로 렌더링한다. 그래서 스크래퍼로 목록을 뽑고,
이름 매핑·강도·향미는 **Claude가 판단**해서 반영한다(자동 differ보다 안전).

| 라인 | cateCd | 앱 compat | nameKo 접미어 |
|------|:--:|--------|--------|
| iperEspresso 전용 | 001006006 | `iperespresso` | (없음) |
| 네스프레소 오리지널 호환 | 001008 | `original` | ` (네스프레소 호환)` |

## ⚠️ illy의 한계
- HTML 스크래핑이라 마크업이 바뀌면 스크래퍼가 깨질 수 있다. 결과를 항상 눈으로 검증.
- 상품명이 지저분하다: `18P 아라비카 셀렉션 캡슐커피 [콜롬비아]`, `에스프레소 호환* 캡슐 [클라시코:미디엄]`.
- 강도·향미(aromatics)는 목록에 없다. **강도·향미·영문명은 illy 공식 상세 이미지(아로마 설명 + 원두 개수)** 로 사람이 채운다.

## 실행 절차
1. **스크래핑**:
   - `node scripts/illy-fetch.mjs iperespresso`
   - `node scripts/illy-fetch.mjs nespresso`
   - 출력 JSON: `{ goodsNo, title, packSize, packPrice, perCapsule, buyUrl }` (번들 자동 제외)
2. **대조**: `src/data/capsules/illy.json`과 비교. 아래 이름 매핑표로 상품명 ↔ nameKo를 맞춘다.
3. **반영**:
   - **신규(매핑표에 없는 새 상품)** → 추가. 가격·구매URL·caffeine은 스크래핑값, **강도·향미·영문명은 공식 이미지로 채움**(사용자에게 이미지 요청). 없으면 강도 0.
   - **사라진 캡슐** → `isEnabled: false` (소프트 단종).
   - **기존** → `perCapsule`로 `price` 갱신, `buyUrl`(goodsNo) 갱신. 큐레이션 값(강도/향미/설명)은 보존.
4. **검증**: `npm run typecheck`. 필요시 preview.
5. **보고**: 추가/단종/가격변경 요약. 요청 없이 커밋·푸시 금지.

## 이름 매핑표 (상품명 → nameKo)
**iperEspresso (compat iperespresso)**
- `18P 캡슐커피 [클라시코:…]` → `클라시코`
- `18P 캡슐커피 [인텐소:…]` → `인텐소`
- `18P 캡슐커피 [포르테]` → `포르테`
- `18P 캡슐커피 [클라시코룽고:…]` → `클라시코 룽고`
- `18P 캡슐커피 [디카프:…]` → `디카페나토`
- `18P 아라비카 셀렉션 캡슐커피 [콜롬비아/과테말라/브라질/에티오피아]` → `아라비카 콜롬비아/과테말라/브라질/에티오피아`
- `18P 필터 캡슐커피 [인텐소:…]` → `필터 인텐소`
- `18P 필터 캡슐커피 [클라시코:…]` → `필터 클라시코`

**네스프레소 호환 (compat original, nameKo에 ` (네스프레소 호환)` 접미)**
- `에스프레소 호환* 캡슐 [클라시코:…]` → `클라시코 (네스프레소 호환)`
- `[인텐소:…]` → `인텐소 (네스프레소 호환)` / `[포르테:…]` → `포르테 (네스프레소 호환)`
- `[클라시코룽고:…]` → `클라시코 룽고 (네스프레소 호환)` / `[인텐소룽고:…]` → `인텐소 룽고 (네스프레소 호환)`
- `[디카프:…]` → `디카페나토 (네스프레소 호환)`

매핑표에 없는 상품명이 나오면 = 신규 라인업이므로 사용자에게 확인하고 매핑을 추가한다.
