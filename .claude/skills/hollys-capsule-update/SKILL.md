# 할리스(HOLLYS) 캡슐·패키지 업데이트

네이버 브랜드스토어 API(`scripts/hollys-fetch.sh`)로 캡슐커피 카테고리를 가져와
`src/data/capsules/hollys.json`(단품, id 8000번대) · `src/data/packages/hollys.json`(패키지, id 9000번대)을 최신화한다.
할리스 캡슐은 전부 **네스프레소 오리지널 호환**(`compat: original`), 브랜드는 `할리스`. 채널 슬러그 `hollys2188`.

## ⚠️ API의 한계와 관문
- **관문**: `User-Agent` + `Referer`(브랜드 경로) 헤더가 둘 다 있어야 200. 루트 referer/헤더 없음 → 429. `pageSize`는 80 이하.
- **강도·향미 미제공** — 이름·가격·이미지·구매URL만 있다.
  - 신규 단품은 강도 0·향미 빈값·영문명 자동으로 추가되고 `[보완 필요]`로 플래그.
  - 공식 제품 정보로 사람이 채운다(수치 없으면 향미 기반 추정 + `desc` 끝에 "…추정치입니다." 안내). 기존 큐레이션 값은 보존.

## 상품 분류 (이름 예: `[할리스] 캡슐커피 {블렌드} 10개입 (네스프레소 오리지널 호환)`)
- **단품**: "종"이 없고 `{블렌드} N개입` → 캡슐(개당가 = 가격 ÷ 개입). 블렌드명은 "[할리스]"·"캡슐커피"·"개입" 사이 텍스트. `디카페인` 포함 시 `caffeine: decaf`.
- **패키지**: `N종`이 있음 → 세트. **자동 반영하지 않는다**(아래 참고).

## ⚠️ 패키지는 "감지 후 물어보기" (중요)
할리스 패키지는 **고정 구성이 아니라 선택형(옵션) 상품**일 수 있고, 이름도 불규칙해(`25+25캡슐 외 4종` 등) 구성·수량을 정확히 뽑기 어렵다.
- 스크립트는 기본적으로 패키지를 **감지만 하고 반영하지 않는다**(`[패키지 감지]`로 목록만 출력).
- **반드시 사용자에게 포함할지 물어본다.** 사용자가 포함을 원할 때만 `--with-packages`로 재실행한다(그때도 수량은 추정치라 `[패키지 수량 확인]`을 사람이 검증).
- 기본(옵션 없이)은 `packages/hollys.json`을 건드리지 않는다. 현재 할리스는 패키지 없이(단품만) 운영한다.

## 실행 절차
1. **미리보기**: `node scripts/update-hollys.mjs --dry-run` → 단품 변경 + `[패키지 감지]` 확인.
2. **패키지 여부 질문**: 감지된 패키지가 있으면 사용자에게 "포함할까요?"를 묻는다. 기본은 제외(선택형 상품).
3. **반영**: 단품만 → `node scripts/update-hollys.mjs` / 패키지도 포함(사용자 동의 시) → `node scripts/update-hollys.mjs --with-packages`
4. **신규 단품 보완**(`[보완 필요]`): 공식 정보로 `name`(영문)·`notes`·`desc`·강도/산미/바디/쓴맛을 채운다. 할리스 공식 스펙시트는 바디·산미·향미·산지를 제공(강도·쓴맛은 없어 추정 + `desc`에 안내). 새 향미는 `src/data/notes.json`에 추가.
5. **(패키지 포함 시) 수량 보정**: `[패키지 수량 확인]`의 실제 구성/수량을 확인해 `packages/hollys.json`의 `items`를 수정.
6. **검증**: `npm run typecheck`(필수), 필요시 preview.
7. **보고**: 단품 추가/단종/가격/보완필요 + 감지된 패키지(포함/제외 결과) 요약 전달.

## 커밋
- **요청 없이 커밋·푸시하지 않는다.**

## 참고
- 단품 가격은 `할인가(benefitsView.discountedSalePrice) ÷ 개입`(개당), 패키지 가격은 할인가 전체.
- 이미지 필드는 `representativeImageUrl`, 구매URL은 `https://brand.naver.com/hollys2188/products/{id}`.
- `data.ts`가 `hollys.json`·`packages/hollys.json`을 다른 브랜드와 합쳐 노출한다. 신규 브랜드 추가 시 `data.ts` import와 `i18n`의 `BRAND_LABELS`(할리스→HOLLYS)도 확장한다.
